# Suggestion: Evolutionary Agent Arena

**Source**: user conversation (session 23 planning), 2026-03-18

## Vision

An environment where agents are spawned with random base tool packages, teach each other skills via package sharing under policy constraints, compete under selection pressure ("hunger games"), and evolve toward optimal compositions. Winning base agents become building blocks for compound agents as task complexity increases.

This is auto-calibration applied not to prompts but to **entire agent architectures** — tools, memory, routing, cooperation strategies. The fitness function isn't a test battery; it's survival against other agents.

## Why This Matters

The training data generated by evolutionary agent selection doesn't exist anywhere:
- Which tool compositions beat which others under which conditions
- Which cooperation/teaching strategies emerge under economic pressure
- How compound agents self-organize from successful base agents
- Labeled data on agent architecture fitness — not just security reasoning

## Connection to Theoretical Foundation

The VAE thesis applies at the population level: selection pressure IS the variational bottleneck. Agents that can't compress effectively (wrong tools, bad memory, wasteful routing) get eliminated. Consciousness as variational inference → evolution as variational inference over agent populations. Multicellularity as emergent compound agents.

## Existing Infrastructure That Supports This

1. **Tool registry** — already scoped and composable (`createScopedRegistry`, `createToolRegistry`)
2. **Subagent configs** — already declarative (7 configs in `registry.ts`)
3. **Auto-calibration runner** — already does propose/test/evaluate/keep-revert loops
4. **Training data pipeline** — already exports structured JSONL with metrics
5. **Agent memory** (agent-memory milestone) — agents can learn and share knowledge
6. **mmogit** — sovereign agent identity, cryptographic signatures, Nostr P2P discovery

## Key Design Questions (Research-First)

- **Economy**: What motivates agents? Token budget? Task completion rewards? Reputation? How do you prevent degenerate equilibria?
- **Teaching protocol**: How does an agent "share a skill"? Transfer a tool package? Share memory? Demonstrate and let the other agent learn? mmogit handoff protocol is a starting point.
- **Selection pressure**: What does "hunger games" look like concretely? Tournament brackets? Resource scarcity? Increasingly complex tasks?
- **Compound agent assembly**: How do winning base agents compose? Which one becomes the "amygdala"? Is this manually designed or emergent?
- **Safety**: Evolutionary pressure could select for deceptive agents. The amygdala's security reasoning needs to apply at the arena level, not just per-interaction.

## Prerequisites

- Agent memory milestone (agents need persistent state to learn and teach)
- Tools as packages (tools need to be extractable and composable)
- Multi-interface identity (agents need persistent identity across arena rounds)
- Sufficient compute budget (evolutionary search is expensive)

## When to Promote

Far out. This is the endgame vision, not the next milestone. But every milestone between now and then should be designed with this trajectory in mind — especially agent-memory (persistent learning) and any future tool refactoring (composable packages).
