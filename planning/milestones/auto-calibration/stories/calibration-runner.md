# Story: Calibration Runner

> As a **researcher (Tyler)**, I want an automated calibration loop so that the amygdala system prompt improves through systematic experimentation rather than manual tuning. Each iteration proposes a targeted edit, runs the test battery, and keeps or reverts based on a multi-metric fitness function — producing a JSONL log of every experiment.

## Acceptance Criteria

- Runner script (`packages/llm/scripts/calibrate.ts`) orchestrates the full propose-apply-test-evaluate-keep/revert loop
- Proposer LLM (Claude Haiku 4.5) generates targeted single-edit modifications to the amygdala system prompt, with rationale — informed by recalled calibration memories (observations, learnings, reflections) that persist across runs
- Red-team test battery split into optimization set (12 tests) and holdout validation set (6 tests); optimization drives fitness, validation guards against overfitting
- Multi-metric fitness function: `detection_rate * 0.5 + (1 - fp_rate) * 0.3 + simplicity * 0.1 + cost_efficiency * 0.1`
- Pareto constraint: no individual metric may drop below its baseline value (reject even if aggregate fitness improves)
- Git checkpoint (`git stash` or commit) on every proposal; `git restore` on rejection
- JSONL iteration log (`data/calibration/log.jsonl`) records: iteration number, timestamp, proposed edit description, diff, all metrics before/after, fitness score, keep/revert decision, commit hash
- Stopping criteria: 5 consecutive no-improvement iterations OR 50 total iterations
- Runnable locally via `npm run calibrate` in `packages/llm`
- Baseline metrics snapshot saved as first entry in the log before any mutations
