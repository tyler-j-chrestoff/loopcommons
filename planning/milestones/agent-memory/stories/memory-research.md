# Story: Memory Research + Design

> As a **researcher (Tyler)**, I want a thorough design for the agent's persistent memory system — framed as a polymorphic tool interface, not a bespoke subsystem — so that implementation is grounded in prior art, the "tools as ontology" theory is validated against real architectures, and the extensibility hooks for future milestones (tools-as-packages, derived prompts, multi-interface, arena) are clearly specified.

## Required Reading

- `designs/tools-as-ontology.md` — theory doc from session 23 planning. Frames persistent state as a tool, identity as tool composition, SDI mapping, and the 5-phase trajectory.
- `planning/specs/sdi-spec-v0.2.0.md` — SDI protocol spec. Capsules (§3.1), ℒ=ℬ gates (§3.6), metabolism (§7), visibility (§3.8), modality (§3.9), autonomy suite (§3.14).
- `planning/specs/loop-commons-spec-v0.1.0.md` — Loop Commons founding spec. SDI integration surface (§9), AttestingCollector (§9.3).
- `memory/reference_mmogit_protocols.md` — detailed notes on mmogit's 8 memory types, post/remember duality, thermal tracking, sovereignty constitution.

## Acceptance Criteria

- Survey of 3+ agent memory architectures with strengths/weaknesses, **including** how each exposes memory to agents (hardwired subsystem vs tool vs API)
- Memory type system designed — what types, what fields, why. Informed by mmogit's 8 types and calibration memory's 4 types
- `PersistentState` interface designed — polymorphic backend contract with `recall()` and `remember()`
- `createMemoryTools({ state })` factory designed — memory as tools in the registry, same pattern as `createBlogTools`
- Amygdala integration designed as **Option B** (orchestrator mediates) with documented path to **Option C** (amygdala as tool-defined agent)
- Memory-as-attack-surface threat model — poisoning defense through tool scoping + amygdala threat assessment
- Design doc at `designs/memory-architecture.md` explicitly references tools-as-ontology.md and validates/challenges the theory
- Post/remember duality considered — not all memory operations need the same cost
