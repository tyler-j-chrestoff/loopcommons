# Story: Alive Memory — Subagent-Driven Writes + Hippocampal Consolidation

> As a **researcher (Tyler)**, I want the agent to decide for itself what's worth remembering — not through deterministic extraction rules, but through identity-grounded judgment — and to consolidate its memories between sessions like a hippocampus replaying the day, so that memory becomes a living system that makes the agent more itself over time.

**Depends on**: soul-and-blog story (the soul document defines what's worth remembering)

## Acceptance Criteria

- Subagents can call `memory_remember` (Option C from design doc §4.7) — tool-level gating checks amygdala threat score before persisting
- `extractMemoryWrites` removed from route.ts — the subagent exercises judgment, the threat score gates
- Hippocampal consolidation: a background pass (triggered periodically or on session end) that reviews recent observations and synthesizes learnings/reflections with evidence chains
- ACC-inspired conflict detection: when a new memory contradicts an existing one, flag it (don't silently overwrite)
- Memory writes carry request metadata from the session (provenance enrichment)
- Red-team: subagent-driven memory poisoning tests (compromised subagent tries to write garbage at low threat — verify gating still works)
- Cross-session: consolidation produces meaningful reflections that improve future recall context
