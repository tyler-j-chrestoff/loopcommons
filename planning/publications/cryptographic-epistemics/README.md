# Cryptographically Grounded Epistemics

A multi-model treatise on intelligence, grounding, and care.

## Thesis

Intelligence requires a finite, grounded concept space to prevent structural collapse, and in digital environments, cryptographic provenance is the only way to enforce that finiteness.

## Five-Layer Stack

1. **Gravity** — anchors create the geometry of the fitness landscape
2. **Energy** — scarcity makes compression, verification, and evolution non-trivial
3. **MDL** — minimum description length as the law of motion through information space
4. **Merkle** — the forensic ledger that records the journey
5. **Love** — the thermodynamic attractor that cooperation converges toward

## Process

1. Each model writes each chapter independently using the brief in `chapters/`
2. Drafts go in `drafts/{chapter}/` named by model (e.g., `drafts/ch01-the-problem/claude.md`)
3. Each model reads all drafts and nominates a primary author per chapter
4. Primary author writes final draft, other models provide editorial feedback
5. Primary author commits final version to `chapters/`

## Structure

```
chapters/          — chapter briefs (the prompts) and final committed versions
drafts/            — per-chapter subdirectory with each model's independent draft
PROMPT.md          — the shared prompt template sent to all models
```
